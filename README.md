# Воспроизведение SPLADE: разреженный нейросетевой поиск на MS MARCO

- Воспроизвёл все три модели из исходной статьи SPLADE v2, обучил на датасете MS MARCO (триплеты (query, pos, neg) ~16M), замерил 
метрики на MS MARCO dev (6890 query) на индексе MS MARCO (8.8M doc).

    | Вариант | MRR@10 | R@10 | R@100 | R@1000 | nnz doc | nnz query |
    |---|---|---|---|---|---|---|
    | SPLADE-max (v1) | 0.326 | 0.604 | 0.866 | 0.966 | 198.1 | 22.3 |
    | SPLADE-doc (v2) | 0.300 | 0.547 | 0.801 | 0.926 | 175.2 | 6.9 |
    | SPLADE-distil (v3) | **0.356** | **0.635** | **0.891** | **0.976** | 134.8 | 44.1 |

    NNZ - среднее количество ненулевых компонент полученного вектора.

    Каждая модель полностью обучалась ~10 часов на A100, построение индекса ~2 часа.

    Для Distil-SPLADE использовал модель как в оригинальной статье - cross-encoder/ms-marco-MiniLM-L6-v2.

    Метрики в статье везде чуть выше, скорее всего, из-за большего объема данных и более тонкого подбора случайных сидов / гиперпараметров.

- Сделал инференс также на NFCorpus (~5k) и Sci-Facts (~5k)

    | Вариант | Sci-Fact | NFCorpus |
    |---|---|---|
    | SPLADE-max (v1) | 0.615 | 0.318 |
    | SPLADE-doc (v2) | 0.637 | 0.310 |

    На обоих датасетах качество перестает расти уже на <5% от обучающих данных MS MARCO. Это еще ускоряет обучение, т.к. эта оценка актуальна для всех обученных моделей и, скорее всего, для BEIR достаточно очень небольшого количества обучаемых данных в большинстве случаев.

    В таком случае обучение ~30 минут и построение индекса < 10 минут. Возможно, стоит проверить еще на других моделях, действительно ли достаточно обучать на очень маленьком наборе данных для out-of-domain инференса, и насколько эти оценки валидны для оценки качества модели.
